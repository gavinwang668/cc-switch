## 概要

CC Switch v3.17.0 はアーキテクチャの重要な拡張です。前バージョンがデータ可搬性と使用量監視に重点を置いていたのに対し、今回のリリースは可用性の境界とプロトコル互換性に焦点を当てています。

まず、**ヘッドレス CLI 管理ツール** `cc-switch-cli` を新規追加しました。これはスタンドアロンの Rust バイナリで、Linux ヘッドレスサーバーやリモートターミナルから CC Switch プロキシサービスを管理できます。GUI や Node.js を必要とせず、GUI と同じコアロジックを共有します。

次に、**コアモジュールの抽出**を行い、データベース初期化やプロバイダー管理など Tauri に依存しないコードを `src-tauri/src/core/` に分離し、CLI と GUI が同じインターフェースを介して再利用できるようにしました。

第三に、**Gemini マルチプロトコルサポート** — Gemini プロバイダーは `gemini_native`（ネイティブ）、`openai_chat`、`openai_responses`、`anthropic` の4つの API フォーマットから選択可能になりました。

最後に、**Claude Desktop に Bedrock プロトコルフォーマット**を追加し、既存の `anthropic`、`openai_chat`、`openai_responses`、`claude` フォーマットと並び、利用可能なバックエンドの幅を広げました。

また、非推奨の MCP API メソッド4つを削除し、コードベースの健全性を維持しています。

## 新機能

### CLI ヘッドレス管理ツール

`cc-switch-cli` は、**ヘッドレス Linux 環境**や**リモートサーバー**で CC Switch プロキシサービスを管理するためのスタンドアロン Rust バイナリです。

利用可能なコマンド：

| コマンド | 説明 |
|----------|------|
| `cc-switch-cli start` | プロキシサーバーを起動（フォアグラウンド）、`Ctrl+C` で終了 |
| `cc-switch-cli status` | プロキシサーバーの状態と PID を表示 |
| `cc-switch-cli list-providers [app]` | プロバイダー一覧を表示 |
| `cc-switch-cli add-provider <app> <id> <name>` | プロバイダーを追加 |
| `cc-switch-cli remove-provider <app> <id>` | プロバイダーを削除 |
| `cc-switch-cli switch-provider <app> <id>` | プロバイダーを切り替え |

ビルド（Node.js / pnpm 不要）：
```bash
cargo build --release --manifest-path src-tauri/Cargo.toml --bin cc-switch-cli
```

### Gemini マルチプロトコルサポート

Gemini プロバイダーは以下の API フォーマットから選択可能：

| フォーマット | 説明 | ユースケース |
|-------------|------|-------------|
| `gemini_native` | Gemini ネイティブ API | Google Gemini 公式 API |
| `openai_chat` | OpenAI Chat Completions | OpenAI 互換ゲートウェイ |
| `openai_responses` | OpenAI Responses API | 新しい OpenAI 互換ゲートウェイ |
| `anthropic` | Anthropic Messages API | Anthropic 互換ゲートウェイ |

### Claude Desktop Bedrock プロトコル

Claude Desktop プロバイダーが `bedrock` API フォーマットをサポート：

| フォーマット | 説明 |
|-------------|------|
| `anthropic` | Anthropic 標準フォーマット |
| `claude` | Claude ネイティブフォーマット |
| `openai_chat` | OpenAI Chat Completions |
| `openai_responses` | OpenAI Responses API |
| `bedrock` | AWS Bedrock プロトコル |

## 変更点

### 非推奨 MCP API の削除

以下の `@deprecated` メソッドを削除しました：

| 削除されたメソッド | 代替メソッド |
|--------------------|-------------|
| `getConfig(app)` | `getAllServers(app)` |
| `upsertServerInConfig(app, server)` | `upsertServer(app, server)` |
| `deleteServerInConfig(app, id)` | `deleteServer(app, id)` |
| `setEnabled(app, id, enabled)` | `upsertServer` の `enabled` フィールドを使用 |

## アップグレード注意事項

### CLI バイナリは個別ビルドが必要

`cc-switch-cli` は GUI インストーラに含まれません。上記の `cargo build` コマンドで個別にビルドしてください。

### Gemini プロバイダーのアップグレード

既存の Gemini プロバイダー設定は影響を受けず、デフォルトで `gemini_native` フォーマットが維持されます。プロトコルを切り替えるにはプロバイダー編集時に新しい API フォーマットを選択してください。

### Claude Desktop プロバイダーのアップグレード

既存の Claude Desktop プロバイダー設定は影響を受けません。`bedrock` フォーマットはプロバイダー追加・編集時に選択可能です。
