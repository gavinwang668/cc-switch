## 概览

CC Switch v3.17.0 是架构层面的一次重要扩展。上一版的重点在于数据可携带性和用量观测，这一版则聚焦于可用性边界和协议兼容性。

首先，新增了**CLI 无头管理工具** `cc-switch-cli`，这是一个独立的 Rust 二进制，可以在 Linux 无头服务器或远程终端上管理 CC Switch 代理服务，不再依赖 GUI 或 Node.js 环境。它支持启动/停止代理、查看状态、管理供应商等核心操作，与 GUI 共享同一套核心逻辑模块。

其次，**提取了核心逻辑模块**，将数据库初始化和供应商管理等不依赖 Tauri 的代码抽离到 `src-tauri/src/core/` 模块中，CLI 和 GUI 通过同一套接口复用，确保两种模式下行为一致。

第三，**Gemini 多协议支持**——Gemini 供应商现在可以选择四种 API 格式：`gemini_native`（原生格式）、`openai_chat`（OpenAI Chat Completions）、`openai_responses`（OpenAI Responses API）和 `anthropic`（Anthropic 格式），轻松对接不同上游网关。

最后，**Claude Desktop 新增 Bedrock 协议格式**，与已有的 `anthropic`、`openai_chat`、`openai_responses` 和 `claude` 格式并列，拓宽了 Claude Desktop 的可用后端范围。

此外，本版清理了 4 个已废弃的 MCP API 方法，保持代码库健康。

## 重点内容

- **CLI 无头管理工具**：新增 `cc-switch-cli` 独立二进制，支持在 Linux 无头环境中通过命令行管理代理服务，核心命令包括 `start`、`status`、`list-providers`、`add-provider`、`remove-provider`、`switch-provider`。
- **核心逻辑模块提取**：新建 `src-tauri/src/core/` 模块，将数据库初始化（`core::init_db`）和供应商管理（`core::provider_manager`）封装为不依赖 Tauri 的可复用接口，为后续更多非 GUI 入口奠定基础。
- **Gemini 多协议支持**：Gemini 供应商现在支持 `gemini_native` / `openai_chat` / `openai_responses` / `anthropic` 四种 API 格式，可在表单中自由选择，后端自动完成请求/响应格式转换。
- **Claude Desktop Bedrock 协议**：Claude Desktop 供应商新增 `bedrock` API 格式选项，与已有的格式并列，满足更多路由场景。
- **废弃 API 清理**：删除了 4 个标记为 `@deprecated` 的 MCP 方法（`getConfig`、`upsertServerInConfig`、`deleteServerInConfig`、`setEnabled`），统一使用 `getAllServers` 等新方法。

## 新功能

### CLI 无头管理工具

`cc-switch-cli` 是一个独立的 Rust 二进制，用于在**无头 Linux 环境**或**远程服务器**上管理 CC Switch 代理服务。它通过 Clap 4.5 提供子命令式 CLI，所有命令共享 `--help` 文档。

可用命令：

| 命令 | 说明 |
|------|------|
| `cc-switch-cli start` | 启动代理服务器（前台运行），支持 `Ctrl+C` 优雅退出 |
| `cc-switch-cli status` | 查看代理服务器运行状态、监听地址和 PID |
| `cc-switch-cli list-providers [app]` | 列出指定应用类型的全部供应商，不指定则列出全部 |
| `cc-switch-cli add-provider <app> <id> <name>` | 添加供应商，支持 `--api-key` 和 `--base-url` 参数 |
| `cc-switch-cli remove-provider <app> <id>` | 删除指定供应商 |
| `cc-switch-cli switch-provider <app> <id>` | 切换当前供应商 |

构建方式（无需 Node.js / pnpm）：
```bash
cargo build --release --manifest-path src-tauri/Cargo.toml --bin cc-switch-cli
```

### 核心逻辑模块提取

将以下逻辑从 Tauri 命令处理函数中提取到独立的 `core` 模块：

- **数据库初始化**：`core::init_db()` 负责创建数据库目录和初始化表结构
- **供应商管理**：`core::provider_manager` 提供 `get_all_providers`、`get_current_provider_id`、`switch_provider`、`remove_provider`、`get_provider_by_id` 等方法

Tauri 命令现在将参数转发给 `core` 模块，CLI 也直接调用 `core` 模块，确保两种入口的行为完全一致。

### Gemini 多协议支持

Gemini 供应商现在可以选择以下 API 格式：

| 格式 | 说明 | 适用场景 |
|------|------|----------|
| `gemini_native` | Gemini 原生 API 格式 | Google Gemini 官方 API |
| `openai_chat` | OpenAI Chat Completions 格式 | OpenAI 兼容网关 |
| `openai_responses` | OpenAI Responses API 格式 | 较新的 OpenAI 兼容网关 |
| `anthropic` | Anthropic Messages API 格式 | Anthropic 兼容网关 |

选择格式后，代理服务器会自动在 Gemini 原生格式和目标格式之间进行请求/响应转换，无需手动处理协议差异。

### Claude Desktop Bedrock 协议

Claude Desktop 供应商新增 `bedrock` API 格式选项，扩展了其可用后端范围：

| 格式 | 说明 |
|------|------|
| `anthropic` | Anthropic 标准格式 |
| `claude` | Claude 自有格式 |
| `openai_chat` | OpenAI Chat Completions |
| `openai_responses` | OpenAI Responses API |
| `bedrock` | AWS Bedrock 协议 |

## 变更

### 废弃 MCP API 移除

下列 `@deprecated` 方法已被删除，请使用对应的新方法：

| 废弃方法 | 替换方法 |
|----------|----------|
| `getConfig(app)` | `getAllServers(app)` |
| `upsertServerInConfig(app, server)` | `upsertServer(app, server)` |
| `deleteServerInConfig(app, id)` | `deleteServer(app, id)` |
| `setEnabled(app, id, enabled)` | 在 `upsertServer` 中设置 `enabled` 字段 |

## 升级提醒

### CLI 二进制需要单独构建

`cc-switch-cli` 不包含在 GUI 安装包中。如需使用 CLI 模式，需用上述 `cargo build` 命令单独构建。

### Gemini 供应商升级

已有 Gemini 供应商配置不受影响，默认为 `gemini_native` 格式。如需切换协议，在编辑供应商时从表单中选择新的 API 格式。

### Claude Desktop 供应商升级

已有 Claude Desktop 供应商配置不受影响。`bedrock` 格式仅在新添加或编辑现有供应商时可选。
