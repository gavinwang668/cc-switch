## Overview

CC Switch v3.17.0 represents a significant architectural expansion. While the previous release focused on data portability and usage observability, this release centers on availability boundaries and protocol compatibility.

First, a new **headless CLI management tool** `cc-switch-cli` — a standalone Rust binary that manages the CC Switch proxy service from Linux headless servers or remote terminals, without requiring a GUI or Node.js. It supports start/stop, status checks, and provider management through the same shared core logic as the GUI.

Second, **core module extraction** — database initialization and provider management logic that doesn't depend on Tauri has been extracted to `src-tauri/src/core/`, allowing both the CLI and GUI to share the same interface and behave identically.

Third, **Gemini multi-protocol support** — Gemini providers can now choose from four API formats: `gemini_native` (native), `openai_chat`, `openai_responses`, and `anthropic`, making it easy to connect to different upstream gateways.

Finally, **Claude Desktop adds the Bedrock protocol format**, joining the existing `anthropic`, `openai_chat`, `openai_responses`, and `claude` formats.

Additionally, 4 deprecated MCP API methods have been removed to keep the codebase healthy.

- **Core Module Extraction**: new `src-tauri/src/core/` module encapsulating database initialization (`core::init_db`) and provider management (`core::provider_manager`) as Tauri-independent reusable interfaces.
- **Gemini Multi-Protocol Support**: Gemini providers now support `gemini_native` / `openai_chat` / `openai_responses` / `anthropic` API formats, selectable in the form with automatic request/response conversion.
- **Claude Desktop Bedrock Protocol**: new `bedrock` API format option for Claude Desktop providers, expanding available routing options.
- **Deprecated API Cleanup**: 4 `@deprecated` MCP methods removed (`getConfig`, `upsertServerInConfig`, `deleteServerInConfig`, `setEnabled`).

## New Features

### CLI Headless Management Tool

`cc-switch-cli` is a standalone Rust binary for managing the CC Switch proxy service in **headless Linux environments** or on **remote servers**. It provides subcommand-style CLI via Clap 4.5, with `--help` documentation for all commands.

Available commands:

| Command | Description |
|---------|-------------|
| `cc-switch-cli start` | Start the proxy server (foreground), graceful shutdown via `Ctrl+C` |
| `cc-switch-cli status` | Show proxy server status, listen address, and PID |
| `cc-switch-cli list-providers [app]` | List all providers for a given app type, or all if not specified |
| `cc-switch-cli add-provider <app> <id> <name>` | Add a provider, with optional `--api-key` and `--base-url` |
| `cc-switch-cli remove-provider <app> <id>` | Delete a provider |
| `cc-switch-cli switch-provider <app> <id>` | Switch to a different provider |

Build (no Node.js / pnpm required):
```bash
cargo build --release --manifest-path src-tauri/Cargo.toml --bin cc-switch-cli
```

### Core Module Extraction

The following logic has been extracted from Tauri command handlers into the standalone `core` module:

- **Database initialization**: `core::init_db()` handles database directory creation and schema initialization
- **Provider management**: `core::provider_manager` exposes `get_all_providers`, `get_current_provider_id`, `switch_provider`, `remove_provider`, `get_provider_by_id`

Tauri commands now delegate to the `core` module, and the CLI directly calls it — both entry points behave identically.

### Gemini Multi-Protocol Support

Gemini providers can now choose from the following API formats:

| Format | Description | Use Case |
|--------|-------------|----------|
| `gemini_native` | Gemini native API format | Google Gemini official API |
| `openai_chat` | OpenAI Chat Completions format | OpenAI-compatible gateways |
| `openai_responses` | OpenAI Responses API format | Newer OpenAI-compatible gateways |
| `anthropic` | Anthropic Messages API format | Anthropic-compatible gateways |

The proxy automatically converts between Gemini native format and the selected target format.

### Claude Desktop Bedrock Protocol

Claude Desktop providers now support the `bedrock` API format:

| Format | Description |
|--------|-------------|
| `anthropic` | Anthropic standard format |
| `claude` | Claude native format |
| `openai_chat` | OpenAI Chat Completions |
| `openai_responses` | OpenAI Responses API |
| `bedrock` | AWS Bedrock protocol |

## Changes

### Deprecated MCP API Removal

The following `@deprecated` methods have been removed:

| Deprecated Method | Replacement |
|-------------------|-------------|
| `getConfig(app)` | `getAllServers(app)` |
| `upsertServerInConfig(app, server)` | `upsertServer(app, server)` |
| `deleteServerInConfig(app, id)` | `deleteServer(app, id)` |
| `setEnabled(app, id, enabled)` | Set `enabled` field in `upsertServer` |

## Upgrade Notes

### CLI Binary Must Be Built Separately

`cc-switch-cli` is not included in the GUI installer. Use the `cargo build` command above to build it separately.

### Gemini Provider Upgrade

Existing Gemini provider configs are unaffected; they default to `gemini_native` format. To switch protocols, edit the provider and select a new API format.

### Claude Desktop Provider Upgrade

Existing Claude Desktop provider configs are unaffected. The `bedrock` format is available when adding or editing providers.
